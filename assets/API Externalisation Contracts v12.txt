Callouts To Clients
1. Clients should implement parsing logic for our API response schema that can handle the addition of new fields without breaking their systems. We guarantee no removal of existing fields; if removal is necessary, we will notify all clients in advance and proceed only after receiving their approval.
2. Our security team will handle IP address whitelisting for all clients. If client’s IP addresses change, they’d need to inform us beforehand.
3. For credential rotations in case of leaks, a hotline communication to be established.
4. Vendors should send ASN quantities in unit sizes (pieces), not case sizes.
5. All write APIs will require an X-Idempotency-Key header to prevent duplicate processing.
6. There is no API for updating an ASN/Invoice. If an update is required, ASN/Invoice will have to be cancelled/deleted using the below api and re-create it again with different invoiceNumber
7. Each API is rate-limited. Currently each api is rate-limited to be 60 RPM per clientId per api
8. Masters to be maintained by Clients
* Vendor-Codes
* Store-Codes
* SKU-Codes
9. Timestamps in the api response will be in UTC time-zone


High level SOP for New onboarding
* Client shares their IP address with Zepto for IP whitelisting without which calls will be blocked
* Zepto will share client-id and client-secret to the clients. Clients will add these values in the api headers while calling zepto api. These two values will help in authentication and authorization of the clients

APIs
1. List PO Events (Released/Updated in Past X days)
This Paginated API, by default, retrieves the latest PO snapshots for all POs created in the last X days, sorted by release date in descending order. It can be used to get the status of all open POs or as a fallback if event-based integration fails. If more detailed information is needed in the response, use the appropriate input parameters as described below
API Headers
Header
	Description
	Content-Type
	application/json — Specifies the request body format.
	X-Client-Id
	Your client ID for authentication (will be shared to you by zepto)
	X-Client-Secret
	Your client secret for authentication (will be shared to you by zepto)
	Input Parameters
Parameter
	Description
	days
	The number of past days (now - days) for which you want to receive PO events. (Maximum: 45 days)
	vendorCodes
	Codes identifying the vendor. This parameter is optional; if not provided, all vendor codes under the client ID will be included in the response. (Maximum: 10 vendor codes)
	poCodes
	The PO Codes for which you want to retrieve event details. This parameter is optional; if not provided, events for all POs under the client ID will be included in the response. (Maximum: 10 PO numbers)
	includeAllPoEvents
	A boolean flag to request all streams of events for PO updates, rather than just the latest PO snapshot. This parameter is optional; the default value is `false`.
	includeLineItemDetails
	A boolean flag to include detailed line items in the response. This parameter is optional; the default value is `false`.
	pageSize
	The number of purchase orders returned per page. This parameter is optional; the default value is `10`, with a maximum of `20`.
	pageNumber
	The page number for pagination. This parameter is optional; the default value is `1`.
	Sample curl
GET /api/v1/external/po/events?days=7&pageSize=10&pageNumber=2&vendorCodes=KK-11111,KK-22222,KK-33333&includeLineItemDetails=true&poNumber=P431909,P431910
Host: silkroute.zeptonow.dev
clientId: client-id
clientSecret: client-secret

* Max size of days , pageSize will be restricted based on estimates and discussions
* client-id and client-secret are static credentials and will be shared with clients at the time of their onboarding




Use eventId as idempotency key to avoid re-processing the same event twice.

EventType Enums
Enum
	Description
	`CreatePO`
	Indicates the initial release of a Purchase Order (PO).
	`UpdatePO`
	Indicates subsequent changes made to an already released PO.
	`CancelPO`
	Indicates that a previously released PO has been cancelled.
	Status Enums
Enum
	Description
	`RELEASED`
	The Purchase Order (PO) demand has been raised.
	`LOCKED`
	At least one invoice or Advance Ship Notice (ASN) has been received for the PO.
	`CANCELLED`
	The previously released PO has been cancelled.
	`EXPIRED`
	The previously released PO has expired.

2.a. ASN Creation API
Zepto exposes this endpoint to inform us about any invoice/asn creation requests made by the vendors for any of our POs
API Path Params


POST /api/v1/external/asn/

API Headers
Header
	Description
	Content-Type
	application/json — Specifies the request body format.
	X-Idempotency-Key
	[string] Unique identifier for the request generated and shared by the client. We will use this to identify duplicate requests (idempotency)
(can be a uuid)
	X-Client-Id
	Your client ID for authentication (will be shared to you by zepto)
	X-Client-Secret
	Your client secret for authentication (will be shared to you by zepto)
	API Body
details of some key-attributes in the api-body
Field
	Type
	Description
	Possible Values/Formats
	invoiceTotals.discountDetails.totalDiscountAmount
	Float
	The total discount amount applied in the invoice as against the base-price value in PO (Mandatory Field, Send 0 if NA)
	A positive decimal number
	invoiceTotals.grandTotalAmount
	Float
	The grand total amount, including taxes, after all discounts have been applied. (Mandatory Field)
	A positive decimal number
	invoiceDetails.invoiceNumber
	String
	The unique identifier for the invoice. (Mandatory Field, unique invoiceId required for each asn request)
	Alphanumeric, no spaces
	purchaseOrderDetails.purchaseOrderNumber
	String
	The unique identifier for the purchase order against which this invoice/asn is created (Mandatory Field)
	Alphanumeric, no spaces
	invoiceFile
	File
	PDF file containing invoice (Optional)
	PDF file containing invoice
	itemDetails[0].productIdentifier.buyerProductIdentifier.skuCode
	String
	The unique article code (sku_id) for the product which zepto understands (Mandatory Field)
	Alphanumeric code
	itemDetails[0].productIdentifier.buyerProductIdentifier.materialCode
	String
	The unique article code for the product which zepto understands (Mandatory Field)
(one of skuCode or materialCode has to be present. Preferably send skuCode. If both are present, skuCode will be used by us)
	Numeric code
	itemDetails[0].mrp
	Float
	MRP of the invoiced quantities
	A positive decimal number
	itemDetails[0].basePrice
	Float
	Base-price (unit-cost, tax-exclusive) of the invoiced quantities
	A positive decimal number
	itemDetails[0].quantity.invoicedQuantity.amount
	Float
	The quantity of the product being invoiced. (Mandatory Field)
	A positive decimal number
	seller.soldFrom.id
	String
	The unique vendor code identifier used by Zepto, representing the vendor who is creating the ASN.
	Alphanumeric

Attribute Name Enums

	Amendment Reason
	MRP
	Request amendment if there is a mismatch between the PO and your master data.
